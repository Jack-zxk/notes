工作区  暂存区  本地仓库  远程仓库

将文件提交到暂存区：
git add 【文件名】 -A 提交所有变化
git add 【文件名】 .  提交新文件和被修改的文件，不包括删除的文件
git add 【文件名】-u 提交被修改和被删除的文件，不包括新文件

将暂存区文件提交到本地仓库：
git commit -m "描述信息" 

将本地仓库中的文件推送到远程仓库中：
git push <远程仓库名> <远程分支名>

将远程仓库中文件拉到工作区中
git pull <远程仓库名> <远程分支名>：<本地分支名>
 ：<本地分支名>可以省略
 
以下相当于git pull 命令：
git fetch  从远程仓库拿到本地仓库
git merge  从本地仓库更新到工作区

查看提交的状态
git status

将文件夹变成git管理的仓库
git init

将本地仓库与远程仓库进行同步：
git remote add 	origin 【github的仓库地址】 //分为ssh和https两种  origin连接远程仓库的别名

删除与远程仓库的连接：
git remote remove origin  

查看提交的历史版本：
git log 

回退到某个版本：
git reset --hard commit_id

分支操作

创建分支：
git branch dev

切换分支：
git checkout dev

查看当前分支：
git branch

合并dev分支到当前分支：
git merge dev

删除分支：
git branch -d dev
